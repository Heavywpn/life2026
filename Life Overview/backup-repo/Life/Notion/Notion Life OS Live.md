    - **1. Setting Up New Projects**
        - I recommend starting in the Dashboard when new to this template: this is where you can set up your Goals, Milestones and - notably for this task manager: Projects can be created ready to add tasks into.
        - Once you have created your Projects you can view them on the [[Projects 2]] Dashboard, or return to the homepage [[Notion Life OS Live]] and manage all tasks from there.
    - **2. Adding New Tasks & Processing Them from The Inbox**
        
        1. Add all the individual tasks you have on your mind to the Tasks inbox on the home screen
            
            ![[Screenshot_2023-03-23_052256.png]]
            
            or on the [[Quick Drop Inbox 2]] page.
            
            ![[Screenshot_2023-03-23_052636.png]]
            
        2. You can now **sort & organise your tasks.**
            1. Use the **daily priority** field if you want them to be in a certain order on the day of delivery
            2. adding a **do date** means they appear in the main task lists, not adding a do date means they will only be visible in the related Project you add them into.
            3. Set a start _and end date_ in the Do Date column if you want a task to last over more than one day on your time lines.
            4. You can link tasks to other databases as needed, and even set a **context** tag so you can e.g. filter a view of tasks by ‚Äòcomputer‚Äô or a person tag like ‚Äòmum‚Äô (think People, Places or Things)
            5. Setting A **Deadline date** means they will also appear in the deadlines views.
                
                ![[Screenshot_2022-02-18_at_16.56.01.png]]
                
            6. Adding them to a **Related Project** in the relation column sorts them into the system ready to go - so input this last.
        3. Once you assign them to a related project, they will clear from the inbox. Remember: They will then be filtered into the Action Dashboard views if you have assigned a ‚Äò**Do on‚Äô** date to them. If they don't have a due date, you can view them in the relevant **project** page. You can also view your master under the ‚öôÔ∏è System toggle if you need to.
            
            ![[Screenshot_2023-03-23_053630.png]]
            
        
          
        
        1. In all task views: entries disappear when the Status Column is set to ‚ÄòDone‚Äô or is checked
            - You can change how you view Status in any view - by selecting properties, Status, and then edit Property - where you can choose Show as to choose if it is displayed as a Checkbox or a Select View
                
                ![[LifeOS_Guide_Set_Task_Status_View.gif]]
                
        2. If you want to more permanently delete completed tasks there is an Archive Task Bin in [[P.A.R.A Dashboard 2]] - this is simply the filtered where ‚ÄòDone‚Äô is checked. Select the all in bulk and delete.
        
        - If you want to share the system with friends, family or colleagues, you can use the Assignee column to Assign Tasks (you‚Äôll need a paid Notion plan to add more team members).
        - Once a task is related to a project, tasks can be viewed within the filtered projects page , and in the master action dashboards on the and the dashboard.
    - **3. Using Sub-Tasks & Dependencies**
        
        Headlines
        
        - Create SubTasks in the Parent Task Page or by creating a new task and then selecting it‚Äôs Parent, or by using the New Task or New Parent & Sub-Task Buttons
        - Think of Parent Tasks as Master tasks with more than one action. Think of the Related Projects as larger projects that contain multiple parent and sub-tasks. E.g. A project might be _‚ÄòMy Hobby‚Äô, ‚ÄòMy Online Business‚Äô, Home Renovation‚Äô_ etc.
        - All new tasks begin as without being blocked or ‚Äòdependent‚Äô on anything else. They are marked as ‚Äòüü¢¬†Actionable‚Äô.
        - If you want a task to be dependent on another (marked as ‚Äòüî¥¬†Waiting‚Äô until another is completed) you can link them in the **Timeline View** but dragging an arrow from one to another, or simply use the ‚Äòblocked by‚Äô or ‚Äòblocking‚Äô column to select dependent tasks.

    - **4. Using Recurring Tasks**
        
        You can create **recurring tasks** in two ways with this system
        
        - **A.** Set up Automated Recurring Tasks using Notions Recurring Database Templates* (Recommended)
            
            Using Notions Recurring Database Templates to set up automatically generated entries is the newest and best way to create recurring tasks in Notion beyond using a 3rd party API workaround.
            
            1. Click on the down arrow next to the ‚Äò==NEW‚Äô== button on any task database view and duplicate the existing the Template called ==‚ÄòDuplicate for New Recurring Task‚Äô==
                
                ![[BC_SC_Notion_05.04.2023_0017812x.png]]
                
            2. Click on the ‚Äò‚Ä¶‚Äô for the new copy, and edit the template to title it and add any pre-filled columns you need (notably give it a related Project)
            3. Click on the ‚Äò‚Ä¶‚Äô again for the task and select the top option ‚ÄòRepeat‚Äô and set the recurrence you want. Use ‚ÄòWeekly‚Äô if you wish the task to generate on selected days of the week.
                
                ![[BC_SC_Notion_05.04.2023_0017792x.png]]

            5. If you want to move a recurring task to a new Do Date once generated - just add a Do Date as normal and it will override the generated date and be moved in the calendar.
            6. **If you want to move a recurring task to a new Do Date** once generated - just add a Do Date as normal and it will override the generated date and be moved in the calendar.
