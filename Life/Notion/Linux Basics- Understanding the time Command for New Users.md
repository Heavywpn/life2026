## Objectives

- Learn how to use the time command in Linux for timing scripts and commands.
- Understand the three modes of time command output: real, user, and system.
- Explore the difference between built-in and external time command versions.
- Learn to use options like -o for output file and -a for append mode.
- Recognize the practical applications of the time command in troubleshooting and system updates.

## Instructions

1. Open your terminal on a Linux system.
2. Type the word `time` followed by any Linux command you want to time.
3. Review the output, which will display real, user, and system time taken to execute the command.
    - Focus on **real time** for practical timing results.
    - Understand that **user mode** refers to the time spent in user-level code execution.
    - Know that **system mode** (or kernel mode) refers to time spent running kernel-level code.
4. To use additional options, type the full path to the time command.
    - Use `o` to write output to a file.
    - Add `a` to append to the file instead of overwriting.
5. Apply the time command to measure durations of interest, such as script execution or software updates.
6. Differentiate between built-in shell time and external time command by using the full path for the latter.
